# Agent Instructions

Before claiming any coding task is done, follow `AIDONE.md`.

Use P0 and P1 by default. Apply P2 when the task touches production, migrations, external services, background jobs, payments, auth, user data, scale, or long-term maintenance.

At the end of coding work, report:

1. Product result.
2. Files changed.
3. AIDONE P0/P1/P2 acceptance status.
4. Verification commands and results.
5. Manual verification steps.
6. Remaining risk.

Do not say "done", "fixed", "works", "ready", or "passes" without fresh verification evidence.
