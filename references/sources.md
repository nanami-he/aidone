# Sources And Related Standards

AIDONE was derived from practical AI coding workflows and common software engineering review criteria, then translated into product-facing acceptance questions.

## Local Source Material Used For The First Draft

The first draft was distilled from local reference repositories and notes:

- `agents.md`: agent-facing repository instruction pattern.
- `agentic-coding`: agent rules, review behaviour, debugging, testing, security, configuration, and completion verification.
- `agent-rules-books`: condensed rules from software engineering books, including simplicity, reversibility, feedback, and legacy-change discipline.
- `cursor-engineering-rules`: practical rules for configuration, testing, security, API design, observability, TypeScript, and review workflows.
- `awesome-cursorrules`: background corpus of AI coding rule templates.

## Why AIDONE Is Not An Awesome List

Awesome lists collect links.

AIDONE defines an acceptance protocol. It can link to references, but the main value is the checklist that an AI coding agent can follow and report against.
