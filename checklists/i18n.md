# i18n And Copy Checklist

Use this when a task adds or changes user-visible text.

## What Counts As User-Facing Text

- Buttons
- Labels
- Headings
- Navigation
- Empty states
- Error messages
- Success messages
- Toasts
- Modals
- Confirmation copy
- Form validation messages
- Notifications
- Emails
- Pricing or plan text
- Legal or consent text
- Onboarding text

## Acceptance Checks

- New user-facing text follows the project's existing i18n, messages, copy, dictionary, locale, or content system.
- If the project has no copy system and the change adds meaningful text, the agent either creates a small central copy structure or explains why this task should not.
- Copy is not scattered through business logic.
- Text that may change by language, region, plan, legal requirement, brand tone, or experiment is not hardcoded deep in components.
- Error and empty-state copy is understandable to users.
- Technical diagnostics are separated from user-facing text.

## User Questions

- List the new user-facing text.
- Where is that text stored?
- If we change language or tone later, do we edit one place or search through components?
- Which text is intentionally hardcoded, and why?
